    • 支持¶

    pgRouting 社区支持可通过 pgRouting 网站、文档、教程、邮件列表等获得。 如果您正在寻求 商业支持，请在下面找到提供 pgRouting 开发和咨询服务的公司列表。

    报告问题¶

    错误在 问题跟踪器 中报告和管理。 请按照以下步骤操作：

    1. 搜索票证以查看您的问题是否已被报告。 如果是这样，请添加您可能发现的任何额外上下文，或者至少表明您也遇到了问题。 这将帮助我们优先考虑常见问题。

    2. 如果您的问题未报告，请为其创建一个`新问题 <https://github.com/pgRouting/pgrouting/issues/new>`__ 。

    3. 在您的报告中包含重现问题的明确说明。 最好的票证包含复制问题所需的确切 SQL。

    4. 如果您可以测试旧版本的 PostGIS 来解决您的问题，请这样做。 在您的票证上，记下问题出现的最早版本。

    5. 对于可以重现问题的版本，请记下操作系统以及 pgRouting、PostGIS 和 PostgreSQL 的版本。

    6. 建议对问题使用以下包装器来查明导致问题的步骤。

    SET client_min_messages TO debug;
  <your code>
SET client_min_messages TO notice;

    邮件列表和 GIS StackExchange¶

    OSGeo 邮件列表服务器上托管有两个 pgRouting 邮件列表：

    • 用户邮件列表：https://lists.osgeo.org/mailman/listinfo/pgrouting-users

    • 开发人员邮件列表： https://discourse.osgeo.org/c/pgrouting/pgrouting-dev/

      • 订阅：https://discourse.osgeo.org/g/pgrouting-dev

    有关如何使用 pgRouting 的一般问题和主题，请写信给用户邮件列表。

    您还可以在 GIS StackExchange 上提问，并使用 pgrouting 标签。查找所有带有 pgrouting 标签的问题，请访问 https://gis.stackexchange.com/questions/tagged/pgrouting 或订阅 pgRouting 问题订阅。
