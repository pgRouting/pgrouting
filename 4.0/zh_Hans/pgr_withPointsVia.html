pgr_withPointsVia¶

pgr_withPointsVia - 经过一系列顶点和/或点的路线。

可用性

Version 4.0.0

  • 函数正式发布。

  • Driving side parameter is positional unnamed and compulsory.

    • Valid values depend on kind of graph

  • Breaking change, signatures no longer available:

    • pgr_withpointsvia(text,text,anyarray,boolean,boolean,boolean,character,boolean)

版本 3.4.0

  • 新提议的函数。

描述¶

给定一个图、图上边缘的一组点以及一个顶点列表，此函数相当于找到 \(vertex_i\) 和 \(vertex_{i+1}\) 之间的最短路径（其中 \(vertex\) 可以是一个顶点或图上的一个点），对于所有的 \(i\) ，满足 \(i < size\_of(via\;vertices)\) 。

路线:

是一个路径序列。

路径:

是路线的一段。

通用算法如下：

  • 用新点构建图表。

    • 点标识符将转换为负值。

    • 顶点标识符将保持正值。

  • 执行 pgr_dijkstraVia - 提议中。

一次通过¶

pgr_withPointsVia(Edges SQL, Points SQL, via vertices, driving side [options])
options: [directed, strict, U_turn_on_edge, details]
返回 (seq, path_id, path_seq, start_vid, end_vid, node, edge, cost, agg_cost, route_agg_cost) 的集合
OR EMPTY SET
示例:

Find the route that visits the vertices \(\{ -6, 15, -1\}\) in that order with right driving side in directed graph..

SELECT * FROM pgr_withPointsVia(
  $$SELECT id, source, target, cost, reverse_cost FROM edges ORDER BY id$$,
  $$SELECT pid, edge_id, side, fraction FROM pointsOfInterest$$,
  ARRAY[-6, 15, -5], 'R');

参数¶

参数

类型

描述

Edges SQL

TEXT

如所述的 SQL 查询。

Points SQL

TEXT

如所述的 SQL 查询。

via vertices

ARRAY [ ANY-INTEGER ]

将要访问的有序顶点标识符数组。

  • 当为正时，它被视为顶点标识符

  • 当负数时，它被认为是一个点标识符

driving side

CHAR

值在 [r, R, l, L, b, B] 中表示驾驶的一侧是：

  • [r, R] 表示右侧驾驶（仅适用于有向图）

  • [l, L]表示左侧驾驶（仅适用有向图）

  • [b, B] 表示两侧通行（仅适用于无向图）

其中：

ANY-INTEGER:

SMALLINT, INTEGER, BIGINT

ANY-NUMERICAL:

SMALLINT, INTEGER, BIGINT, REAL, FLOAT

在 Points SQL 中使用 pgr_findCloseEdges 函数¶

从顶点 \(1\) 开始，按照距离图形最近的顺序访问点 (2.9, 1.8) 上的两个位置。

SELECT * FROM pgr_withPointsVia(
  $e$ SELECT * FROM edges $e$,
  $p$ SELECT edge_id, round(fraction::numeric, 2) AS fraction, side
      FROM pgr_findCloseEdges(
        $$SELECT id, geom FROM edges$$,
        (SELECT ST_POINT(2.9, 1.8)),
        0.5, cap => 2)
  $p$,
  ARRAY[1, -1, -2], 'r', details => true);

   2 |       1 |        2 |         1 |      -1 |    3 |    7 |    1 |        1 |              1

   4 |       1 |        4 |         1 |      -1 |   -2 |    8 |  0.1 |      2.9 |            2.9

   6 |       1 |        6 |         1 |      -1 |   16 |   16 |    1 |        4 |              4
   7 |       1 |        7 |         1 |      -1 |   15 |    3 |    1 |        5 |              5

节点、点的"通过"或"访问"状态。¶

SELECT seq, node,
CASE WHEN edge = -1 THEN $$visits$$
ELSE $$passes in front$$
END as status
FROM  pgr_withPointsVia(
  $$SELECT id, source, target, cost, reverse_cost FROM edges ORDER BY id$$,
  $$SELECT pid, edge_id, side, fraction FROM pointsOfInterest$$,
  ARRAY[-6, 7, -4, 8, -2], 'r', details => true)
WHERE agg_cost <> 0 or seq = 1;
