pgr_depthFirstSearch - 提议中¶

pgr_depthFirstSearch — 返回图的深度优先搜索遍历。 该图可以是有向图或无向图。

Proposed

警告

下一版本的提议功能。

  • 它们并未正式出现在当前版本中。

    • Code has been reviewed therefore is not experimental.

    • Name, signature and functionality might not change.

    • pgTap 测试已经完成。 但可能需要更多。

    • 文档可能需要完善。

可用性

Version 4.0.0:

  • 将输出列标准化为 (seq, depth, start_vid, pred, node, edge, cost, agg_cost)

版本 3.3.0

  • 函数已升级为提议。

版本3.2.0

  • 新实验性功能。

描述¶

深度优先搜索算法是一种遍历算法，从根顶点开始，尽可能深入探索，一旦到达一个没有相邻顶点或所有相邻顶点都已访问的顶点，就会回溯。遍历会一直持续，直到从根顶点可达的所有顶点都被访问。

主要特点是：

  • 该实现适用于 有向 图和 无向 图。

  • 提供从根顶点或一组根顶点开始的深度优先搜索遍历顺序。

  • 可选的非负最大深度参数，用于限制结果的深度到特定的深度。

  • 为了优化的目的，Root vids 中的任何重复值都会被忽略。

  • 它不会生成从根顶点到目标顶点的最短路径。

  • 不保证遍历的总成本最小。

  • 返回值按 start_vid 升序排列。

  • 深度优先搜索运行时间： \(O(E + V)\)

多个顶点¶

pgr_depthFirstSearch(Edges SQL, root vids, [options])
options: [directed, max_depth]
返回集合 (seq, depth, start_vid, pred, node, edge, cost, agg_cost)

参数¶

参数

类型

描述

Edges SQL

TEXT

Edges SQL 如下所述。

root vid

BIGINT

树的根顶点的标识符。

  • 当值为 \(0\) 时，则获取从森林中每棵树的随机节点开始的生成森林。

root vids

ARRAY [ ANY-INTEGER ]

根顶点的标识符数组。

  • \(0\) 值被忽略

  • 出于优化目的，任何重复的值都将被忽略。

其中：

ANY-INTEGER:

SMALLINT, INTEGER, BIGINT

ANY-NUMERIC:

SMALLINT, INTEGER, BIGINT, REAL, FLOAT, NUMERIC

可选参数¶

列

类型

默认

描述

directed

BOOLEAN

true

  • 当 true 时，该图被视为有 有向

  • 如果为 false ，则该图被视为 无向 。

参数

类型

默认

描述

BIGINT

\(9223372036854775807\)

树深度的上限。

  • 为负数时抛出错误。

其他示例¶

示例:

与 单个顶点 相同，但边按 id 降序排列。
