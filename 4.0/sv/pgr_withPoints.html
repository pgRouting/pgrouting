pgr_medPunkter¶

pgr_withPoints - Returnerar den kortaste vägen i en graf med ytterligare tillfälliga toppar.

Tillgänglighet

Version 4.0.0

  • Funktion befordrad till tjänsteman.

  • Driving side parametern är positionell, namnlös och obligatorisk.

    • Giltiga värden beror på typ av graf

  • Utdatakolumner standardiserade till (seq, path_seq, start_vid, end_vid, node, edge, cost, agg_cost)

  • Förändring på gång, signaturer inte längre tillgängliga:

    • pgr_withpoints(text,text,anyarray,anyarray,boolean,character,boolean)

    • pgr_withpoints(text,text,anyarray,bigint,boolean,character,boolean)

    • pgr_withpoints(text,text,bigint,anyarray,boolean,character,boolean)

    • pgr_withpoints(text,text,bigint,bigint,boolean,character,boolean)

    • pgr_withpoints(text,text,text,boolean,character,boolean)

Version 3.2.0

  • Nytt förslag till signatur:

    • pgr_withPoints(Combinations)

Version 2.2.0

  • Ny föreslagen funktion.

Beskrivning¶

Modifiera grafen så att den innehåller punkter definierade av Points SQL. Använd Dijkstra-algoritmen för att hitta den kortaste vägen.

De viktigaste egenskaperna är:

  • Processen utförs endast på kanter med positiva kostnader.

  • Grafens hörn är:

    • positiv när den tillhör Kanter SQL

    • negativ när den tillhör Points SQL

  • Värden returneras när det finns en sökväg.

    • När startvertex och slutvertex är desamma finns det ingen väg.

      • Agg_cost i de icke inkluderade värdena ”v, v” är ”0”

    • När startvertex och slutvertex är olika och det inte finns någon väg:

      • Agg_cost i de icke inkluderade värdena (u, v) är \(\infty\)

  • För optimeringsändamål ignoreras alla duplicerade värden i inmatningsmatriserna för start vids eller end vids eller.

  • De returnerade värdena är ordnade:

    • start_vid stigande

    • end_vid stigande

  • Drifttid: \(O(|start\_vids|\times(V \log V + E))\)

CHAR

Värde i [r, R, l, L, b, B] som anger om förarsidan är:

  • For directed graph:

    • [r, R] for right driving side

  • For undirected graph:

    • [b, B] for both driving sides

  • Default values when parameter is missing:

    • [r, R] for directed graph

    • [l, L] for undirected graph

Använd pgr_findCloseEdges i Points SQL.¶

Hitta vägarna från toppunkten \(1\) till de två närmaste platserna på grafen för punkten (2,9, 1,8).

SELECT * FROM pgr_withPoints(
  $e$ SELECT * FROM edges $e$,
  $p$ SELECT edge_id, round(fraction::numeric, 2) AS fraction, side
      FROM pgr_findCloseEdges(
        'SELECT id, geom FROM edges',
        (SELECT ST_POINT(2.9, 1.8)),
        0.5, cap => 2)
  $p$,
  1, ARRAY[-1, -2]);
