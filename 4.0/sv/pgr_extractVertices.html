pgr_extractVertices¶

pgr_extractVertices - Extraherar information om hörnpunkter

Tillgänglighet

Version 3.8.0

  • Justering av felmeddelanden.

  • Funktion befordrad till tjänsteman.

Version 3.3.0

  • Funktion befordrad till föreslagen.

Version 3.0.0

  • Ny experimentell funktion.

Beskrivning¶

Detta är en hjälpfunktion för att extrahera toppunktsinformationen för en grafs uppsättning kanter.

  • När kantidentifieraren anges kommer den också att beräkna in- och utkanterna

Boost-graf inuti

Underskrifter¶

pgr_extractVertices(Kanter SQL, [dryrun])
RETURNS SETOF (id, in_edges, out_edges, x, y, geom)
ELLER TOM UPPSÄTTNING

När linjegeometrin är känd¶

Kolumn

Typ

Beskrivning

BIGINT

(Valfritt) identifierare för kanten.

LINESTRING

Geometri för kanten.

Denna inre fråga har företräde framför de två följande inre frågorna, och därför ignoreras andra kolumner när kolumnen geom visas.

    • startpunkt

    • endpoint

    • mål

Utförande av provkörning¶

Använd dryrun := true för att få den fråga som genereras för att få fram toppunktsinformationen.

Resultaten kan användas som baskod för att göra en förfining baserad på behoven för backend-utvecklingen.

SELECT  * FROM pgr_extractVertices(
  'SELECT id, geom FROM edges',
  dryrun => true);
NOTICE:
        WITH

        main_sql AS (
          SELECT id, geom FROM edges
        ),

        the_out AS (
          SELECT id::BIGINT AS out_edge, ST_StartPoint(geom) AS geom
          FROM main_sql
        ),

        agg_out AS (
          SELECT array_agg(out_edge ORDER BY out_edge) AS out_edges, ST_x(geom) AS x, ST_Y(geom) AS y, geom
          FROM the_out
          GROUP BY geom
        ),

        the_in AS (
          SELECT id::BIGINT AS in_edge, ST_EndPoint(geom) AS geom
          FROM main_sql
        ),

        agg_in AS (
          SELECT array_agg(in_edge ORDER BY in_edge) AS in_edges, ST_x(geom) AS x, ST_Y(geom) AS y, geom
          FROM the_in
          GROUP BY geom
        ),

        the_points AS (
          SELECT in_edges, out_edges, coalesce(agg_out.geom, agg_in.geom) AS geom
          FROM agg_out
          FULL OUTER JOIN agg_in USING (x, y)
        )

        SELECT row_number() over(ORDER BY ST_X(geom), ST_Y(geom)) AS id, in_edges, out_edges, ST_X(geom), ST_Y(geom), geom
        FROM the_points;
 id | in_edges | out_edges | x | y | geom

Kontrollera att databasen inte har vertices_table¶

DROP TABLE IF EXISTS vertices_table;
NOTICE:  table "vertices_table" does not exist, skipping
DROP TABLE

Rensa upp kolumnerna i den routningstopologi som ska skapas¶

UPDATE edges
SET source = NULL, target = NULL,
x1 = NULL, y1 = NULL,
x2 = NULL, y2 = NULL;
UPDATE 18

Skapa en tabell över hörnpunkter¶

  • När LINESTRING har en SRID används geom::geometry(POINT, <SRID>)

  • För bord med stor kant som är förberedda,

    • Skapa den som UNLOGGED och

    • Efter att tabellen har skapats ALTER TABLE .. SET LOGGED

SELECT  * INTO vertices_table
FROM pgr_extractVertices('SELECT id, geom FROM edges ORDER BY id');
SELECT 17

Skapa routningstopologin på edge-tabellen¶

Uppdatering av informationen om källan

WITH
out_going AS (
  SELECT id AS vid, unnest(out_edges) AS eid, x, y
  FROM vertices_table
)
UPDATE edges
SET source = vid, x1 = x, y1 = y
FROM out_going WHERE id = eid;
UPDATE 18

Uppdatering av informationen om ”målet

WITH
in_coming AS (
  SELECT id AS vid, unnest(in_edges) AS eid, x, y
  FROM vertices_table
)
UPDATE edges
SET target = vid, x2 = x, y2 = y
FROM in_coming WHERE id = eid;
UPDATE 18
