pgr_medPointsDD¶

pgr_withPointsDD - Returnerar köravståndet från en startpunkt.

Tillgänglighet

Version 4.0.0

  • Funktion befordrad till tjänsteman.

  • Förändring på gång, signaturer inte längre tillgängliga:

    • pgr_withpointsdd(text,text,bigint,double precision,boolean,character,boolean)

    • pgr_withpointsdd(text,text,anyarray,double precision,boolean,character,boolean,boolean)

Version 3.6.0

  • Utgångskolumner standardiserade till (seq, depth, start_vid, pred, node, edge, cost, agg_cost)

  • Driving side parametern är positionell, namnlös och obligatorisk.

    • Giltiga värden beror på typ av graf

  • När details är false:

    • Punkter som nås inom avståndet ingår inte.

  • Föråldrade signaturer:

    • pgr_withpointsdd(text,text,bigint,double precision,boolean,character,boolean)

    • pgr_withpointsdd(text,text,anyarray,double precision,boolean,character,boolean,boolean)

Version 2.2.0

  • Ny föreslagen funktion.

Beskrivning¶

Modifiera grafen så att den inkluderar punkter och använd Dijkstra-algoritmen för att extrahera alla noder och punkter som har kostnader som är mindre än eller lika med värdet **avstånd** från startpunkten. De extraherade kanterna kommer att överensstämma med motsvarande spännträd.

Boost-graf inuti

    • [l, L] for left driving side

  • For undirected graph:

  • Default values when parameter is missing:

    • [r, R] for directed graph

    • [l, L] for undirected graph

Använd pgr_findCloseEdges i Points SQL.¶

Hitta köravståndet från de två närmaste platserna på grafen för punkten (2,9, 1,8).

SELECT * FROM pgr_withPointsDD(
  $e$ SELECT * FROM edges $e$,
  $p$ SELECT edge_id, round(fraction::numeric, 2) AS fraction, side
      FROM pgr_findCloseEdges(
        $$SELECT id, geom FROM edges$$,
        (SELECT ST_POINT(2.9, 1.8)),
        0.5, cap => 2)
  $p$,
  ARRAY[-1, -2], 2.3, 'r',
  details => true);
