pgr_contractionDeadEnd - Förslag¶

pgr_contractionDeadEnd - Utför grafkontraktion och returnerar de kontraherade topparna och kanterna.

Föreslagen

Varning

Föreslagna funktioner för nästa version av Mayor.

  • De finns inte officiellt med i den aktuella versionen.

    • Koden har granskats och är därför inte experimentell.

    • Namn, signatur och funktionalitet kanske inte ändras.

    • pgTap-tester har gjorts. Men det kan behövas fler.

    • Dokumentationen kan behöva förbättras.

Tillgänglighet

  • Version 3.8.0

    • Ny föreslagen funktion.

Beskrivning¶

Kontraktion minskar storleken på grafen genom att ta bort några av topparna och kanterna och till exempel lägga till kanter som representerar en sekvens av ursprungliga kanter, vilket minskar den totala tid och det utrymme som används i grafalgoritmer.

De viktigaste egenskaperna är:

  • Processen utförs endast på kanter med positiva kostnader.

  • Returnerar inte hela den kontraherade grafen.

    • Endast ändringar i diagrammet returneras.

  • De returnerade värdena inkluderar:

    • De nya kanterna som genereras av linjär sammandragning.

    • De modifierade topparna som genereras av sammandragning av återvändsgränder.

  • De returnerade värdena är ordnade enligt följande:

    • kolumn id stigande när det är en modifierad vertex.

    • kolumn id med negativa tal fallande när det är en ny kant.

En nod anses vara en återvändsgrändsnod när:

  • Om odelade grafer:

    • Antalet angränsande hörn är 1.

  • Om riktade grafer:

    • När det bara finns en angränsande toppunkt eller

    • När alla kanter är inkommande oavsett antalet angränsande toppar.

Steg för steg återvändsgränd sammandragning¶

Sammandragningen av återvändsgränder kommer att upphöra tills det inte finns några fler återvändsgrändsnoder. Till exempel, från följande graf där \(3\) är återvändsgränden:

Efter kontrakteringen av \(3\) är noden \(2\) nu en återvändsgränd och kontrakteras:

Efter att ha dragit in \(2\), sluta. Noden \(1\) har information om de noder som kontrakterades.

SELECT * FROM pgr_contractionDeadEnd(
$$SELECT * FROM (VALUES
  (1, 1, 2, 1, -1),
  (2, 2, 3, 1, -1),
  /* Rest of the graph */
  (3, 1, 25, 1, 1), (4, 1, 26, 1, 1),
  (5, 25, 25, 1, 1), (6, 25, 26, 1, 1)) AS edges(id,source,target,cost,reverse_cost)$$,
  directed => true);
 type | id | contracted_vertices | source | target | cost
------+----+---------------------+--------+--------+------
 v    |  1 | {2,3}               |     -1 |     -1 |   -1
(1 row)

Steg för att skapa den avtalade grafen¶

Lägg till ytterligare kolumner.

ALTER TABLE vertices ADD is_contracted BOOLEAN DEFAULT false;
ALTER TABLE
ALTER TABLE vertices ADD contracted_vertices BIGINT[];
ALTER TABLE

Spara resultaten i en tabell.

SELECT * INTO contraction_results
FROM pgr_contractionDeadEnd(
  'SELECT id, source, target, cost, reverse_cost FROM edges',
  directed => false);
SELECT 5

Använd kolumnen is_contracted för att ange de toppar som är kontraherade.

UPDATE vertices
SET is_contracted = true
WHERE id IN (SELECT  unnest(contracted_vertices) FROM  contraction_results);
UPDATE 6

Fyll i contracted_vertices med informationen från de resultat som hör till topparna.

UPDATE vertices
SET contracted_vertices = contraction_results.contracted_vertices
FROM contraction_results
WHERE type = 'v' AND vertices.id = contraction_results.id;
UPDATE 5

De kontraherade topparna är inte en del av den kontraherade grafen.

SELECT id, is_contracted
FROM vertices WHERE is_contracted ORDER BY id;

Den kontraherade grafen¶

DROP VIEW IF EXISTS contracted_graph;
NOTICE:  view "contracted_graph" does not exist, skipping
DROP VIEW
CREATE VIEW contracted_graph AS
WITH
vertices_in_graph AS (
  SELECT id FROM vertices WHERE is_contracted = false
)
SELECT id, source, target, cost, reverse_cost
FROM edges
WHERE source IN (SELECT * FROM vertices_in_graph)
AND target IN (SELECT * FROM vertices_in_graph)
ORDER BY id;
CREATE VIEW

Används när avgång/destination inte finns i den avtalade grafen¶

SELECT * FROM pgr_dijkstra(
  'WITH cul_de_sac AS (
    SELECT contracted_vertices || id::INTEGER as v
    FROM vertices WHERE 1 = ANY(contracted_vertices))
  SELECT id, source, target, cost, reverse_cost FROM edges, cul_de_sac
  WHERE source = ANY(v) AND target = ANY(v)

  UNION

  SELECT id, source, target, cost, reverse_cost FROM contracted_graph',
  1, 17);
