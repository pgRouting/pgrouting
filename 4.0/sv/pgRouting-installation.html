Installation¶

Innehåll

Instruktioner för nedladdning och installation av binärfiler för olika operativsystem, ytterligare anmärkningar och korrigeringar som inte ingår i denna dokumentation finns i Installation wiki

För att använda pgRouting måste PostGIS vara installerat, läs informationen om installation i denna installationsguide <https://www.postgis.us/presentations/postgis_install_guide_22.html>`__

Kort version¶

Extrahera tar-bollen

tar xvfz pgrouting-4.0.0.tar.gz
cd pgrouting-4.0.0

För att kompilera förutsatt att du har alla beroenden i din sökväg:

mkdir build
cd build
cmake  ..
make
sudo make install

När pgRouting har installerats måste det aktiveras i varje enskild databas som du vill använda det i.

createdb routing
psql routing -c 'CREATE EXTENSION pgRouting CASCADE'

Hämta källorna¶

Den senaste versionen av pgRouting finns på https://github.com/pgRouting/pgrouting/releases/latest

För att ladda ner denna release:

wget -O pgrouting-4.0.0.tar.gz \
   https://github.com/pgRouting/pgrouting/archive/v4.0.0.tar.gz

Gå till Kort version för mer instruktioner om hur du extraherar tar ball och kompilerar pgRouting.

git

Så här laddar du ner arkivet

git clone git://github.com/pgRouting/pgrouting.git
cd pgrouting
git checkout v4.0.0

Gå till Kort version för mer instruktioner om hur du kompilerar pgRouting (det finns ingen tarball inblandad när du laddar ner pgRouting-arkivet från GitHub).

Aktivering och uppgradering i databasen¶

Aktivering av databasen

pgRouting är en PostgreSQL-förlängning och beror på PostGIS för att tillhandahålla funktioner till slutanvändaren. Nedanstående kod demonstrerar aktivering av PostGIS och pgRouting i databasen.

CREATE EXTENSION postgis;
CREATE EXTENSION pgrouting;

Kontroll av PostGIS- och pgRouting-version efter att de har aktiverats i databasen.

SELECT PostGIS_full_version();
SELECT * FROM pgr_version();

Uppgradering av databasen

Använd följande kommando för att uppgradera pgRouting i databasen till version 4.0.0:

ALTER EXTENSION pgrouting UPDATE TO "4.0.0";

Mer information finns på https://www.postgresql.org/docs/current/sql-createextension.html

Beroenden¶

Beroenden för kompilering

För att kunna kompilera pgRouting måste du se till att följande beroenden är uppfyllda:

  • C- och C++0x-kompilatorer

    • Kompilering med Boost 1.56 upp till Boost 1.74 kräver C++-kompilator med standardstöd för C++03 eller C++11

    • Kompilering med Boost 1.75 kräver C++-kompilator med stöd för C++14-standarden

  • Postgresql-version >= 13

  • Boost Graph Library (BGL) >= 1.56.0

  • CMake >= 3.12

För användardokumentation

  • Sphinx >= 4.0

För dokumentation för utvecklare

  • Doxygen >= 1.7

För testning

  • pgtap

  • pg_prove

För användning:

  • PostGIS version >= 3.0.0

Exempel: Installera beroenden på Linux

Installera kompileringsberoendena

Bygg beroenden

sudo apt install \
   build-essential \
   libboost-graph-dev \
   postgresql-13 \
   postgresql-server-dev-13 \
   postgresql-13-postgis

Valfria beroenden

För dokumentation

sudo apt-get install -y  \
   python3-sphinx \
   python3-sphinx-bootstrap-theme \
   texlive \
   doxygen

För testning

sudo apt install \
   libtap-parser-sourcehandler-pgtap-perl
   postgresql-13-pgtap

Konfigurering¶

pgRouting använder cmake-systemet för att göra konfigurationen.

Konfigurerbara variabler¶

För att se vilka variabler som kan konfigureras

mkdir build
cd build
cmake -L ..

Byggkatalogen skiljer sig från källkatalogen

Konfigurera dokumentationen

Användar- och utvecklardokumentation byggs inte om förutsättningarna inte hittas.

Det mesta av arbetet med dokumentationen har lagts på HTML-filerna. Några variabler för att bygga dokumentation:

Variabel

Standard

Kommentar

BUILD_HTML

BOOL=ON

Om ON, aktivera/avaktivera HTML-skapande för användardokumentation

BYGG_DOXY

BOOL=ON

Om ON, aktivera/avaktivera HTML-skapande för utvecklarens dokumentation

BUILD_LATEX

BOOL=OFF

Om ON, slå på/av byggnad PDF

BYGGHANDLARE

BOOL=OFF

Om ON, aktivera/avaktivera byggnadens MAN-sidor

DOC_ANVÄNDA_BOOTSTRAP

BOOL=OFF

Om PÅ, använd sphinx-bootstrap för HTML-sidor i användardokumentationen

EN

BOOL=ON

om OFF den engelska dokumentationen kommer inte att byggas

ES

BOOL=ON

om OFF den spanska dokumentationen kommer inte att byggas

SV

BOOL=ON

om OFF kommer den svenska dokumentationen inte att byggas

ZH_HANS

BOOL=ON

om OFF den kinesiska förenklade dokumentationen kommer inte att byggas

Byggnad¶

Använda make för att bygga koden och dokumentationen

Följande instruktioner startar från stigen/till/pgrouting/build

$ make          # default build
$ make doc      # build only the user's documentation
$ make doxy     # build only the developer's documentation
$ make all      # build both the code and the user's documentation

Vi har testat på flera plattformar, För installation eller ominstallation behövs alla steg.

Linux

Följande instruktioner börjar från path/to/pgrouting

mkdir build
cd build
cmake  ..
make
sudo make install

Testar¶

För närvarande finns det ingen make test och testning görs på följande sätt

Följande instruktioner börjar från path/to/pgrouting

tools/testers/doc_queries_generator.pl
createdb -U <user> ___pgr___test___
sh ./tools/testers/pg_prove_tests.sh <user>
dropdb -U <user> ___pgr___test___
