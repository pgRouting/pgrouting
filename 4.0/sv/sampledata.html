Huvudgraf¶

En graf består av en uppsättning kanter och en uppsättning hörn.

Följande stad ska läggas in i databasen:

Information som är känd vid denna tidpunkt är geometrin för kanterna, kostnadsvärden, kapacitetsvärden, kategorivärden och vissa platser som inte finns i grafen.

Processen för att få en fungerande topologi börjar med att lägga in kanterna. Efter det beräknas allt annat.

Kanter¶

Databasdesignen för dokumentationen av pgRouting, håller i samma rad 2 segment, ett i geometrins riktning och det andra i motsatt riktning. Därför har viss information prefixet reverse_ som motsvarar segmentet i geometrins motsatta riktning.

Kolumn

Beskrivning

id

En unik identifierare.

Källa

Identifierare av startpunkten för geometrin geom.

mål

Identifierare för geometrins slutpunkt geom

cost

Kostnad för att gå från källa till mål.

omvända_kostnader

Kostnad för att gå från mål till källa.

Kapacitet

Flödeskapacitet från källa till mål.

omvänd_kapacitet

Flödeskapacitet från mål till källa.

category

Flödeskapacitet från mål till källa.

reverse_category

Flödeskapacitet från mål till källa.

x1

\(x\) coordinate of the starting vertex of the geometry.

  • För enkelhetens skull sparas den i tabellen, men kan beräknas som ST_X(ST_StartPoint(geom)).

y2

\(y\) coordinate of the ending vertex of the geometry.

  • För enkelhetens skull sparas den i tabellen, men kan beräknas som ST_Y(ST_EndPoint(geom)).

geom

Segmentens geometri.

 
CREATE TABLE edges (
    id BIGSERIAL PRIMARY KEY,
    source BIGINT,
    target BIGINT,
    cost FLOAT,
    reverse_cost FLOAT,
    capacity BIGINT,
    reverse_capacity BIGINT,
    x1 FLOAT,
    y1 FLOAT,
    x2 FLOAT,
    y2 FLOAT,
    geom geometry
);
CREATE TABLE

Börjar på PostgreSQL 12:

...
x1 FLOAT GENERATED ALWAYS AS (ST_X(ST_StartPoint(geom))) STORED,
y1 FLOAT GENERATED ALWAYS AS (ST_Y(ST_StartPoint(geom))) STORED,
x1 FLOAT GENERATED ALWAYS AS (ST_X(ST_EndPoint(geom))) STORED,
y1 FLOAT GENERATED ALWAYS AS (ST_Y(ST_EndPoint(geom))) STORED,
...

Eventuellt kan index på olika kolumner skapas. Rekommendationen är att ha

  • id indexerad.

  • kolumnerna källa och mål indexerade för att snabba upp pgRouting-frågor.

  • geom indexerad för att påskynda geometriska processer som kan behövas i frontend.

I det här lilla exemplet hoppas indexen över, utom för id

Topologin¶

Denna fråga baseras på toppdata och skapar en topologi genom att fylla i kolumnerna ”källa” och ”mål” i tabellen ”kanter”.

/* -- set the source information */
UPDATE edges AS e
SET source = v.id, x1 = x, y1 = y
FROM vertices AS v
WHERE ST_StartPoint(e.geom) = v.geom;
UPDATE 18
/* -- set the target information */
UPDATE edges AS e
SET target = v.id, x2 = x, y2 = y
FROM vertices AS v
WHERE ST_EndPoint(e.geom) = v.geom;
UPDATE 18

Punkter utanför grafen¶

Attraktioner¶

Ibland fungerar applikationerna ”on the fly” med utgångspunkt från en plats som inte är ett vertex i grafen. Dessa platser kallas i pgRrouting för intressepunkter.

Den information som behövs i intressepunkterna är pid, edge_id, side, fraction.

På denna dokumentation kommer det att finnas cirka 6 fasta punkter av intresse och de kommer att lagras på ett bord.

Kolumn

Beskrivning

pid

En unik identifierare.

Kedge_id

Identifierare för det närmaste segmentet.

sida

Är det på vänster, höger eller båda sidor av segmentet edge_id.

fraktion

Var i segmentet är punkten belägen.

geom

Punkternas geometri.

avstånd

Avståndet mellan geom och segmentet edge_id.

kant

Ett segment som förbinder punktens geom med den närmaste punkten på segmentet edge_id.

nyaPunkt

En punkt på segmentet edge_id som ligger närmast geom.

 
CREATE TABLE pointsOfInterest(
    pid BIGSERIAL PRIMARY KEY,
    edge_id BIGINT,
    side CHAR,
    fraction FLOAT,
    distance FLOAT,
    edge geometry,
    newPoint geometry,
    geom geometry);

Intressanta platser fylls upp¶

Infoga intressepunkterna.

INSERT INTO pointsOfInterest (geom) VALUES

Fyller resten av bordet.

UPDATE pointsofinterest SET
  edge_id = poi.edge_id,
  side =  poi.side,
  fraction = round(poi.fraction::numeric, 2),
  distance = round(poi.distance::numeric, 2),
  edge = poi.edge,
  newPoint = ST_EndPoint(poi.edge)
FROM (
  SELECT *
  FROM pgr_findCloseEdges(
    $$SELECT id, geom FROM edges$$,(SELECT array_agg(geom) FROM pointsOfInterest), 0.5) ) AS poi
WHERE pointsOfInterest.geom = poi.geom;

Någon annan ytterligare modifiering: I denna manual kan punkten \(6\) nås från båda sidor.

UPDATE pointsOfInterest SET side = 'b' WHERE pid = 6;

Kombinationer¶

För att underlätta denna dokumentation kommer vissa kombinationer att lagras på en tabell:
