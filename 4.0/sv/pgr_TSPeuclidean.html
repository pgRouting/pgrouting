pgr_TSPeuclidean¶

  • pgr_TSPeuclidean - Approximation med hjälp av metrisk algoritm.

Tillgänglighet:

Version 4.0.0

  • Resultaten ändras beroende på inmatningsordning

  • Förändring på gång, signaturer inte längre tillgängliga:

    • pgr_tspeuclidean(text,bigint,bigint,double precision,integer,integer, integer,double precision,double precision,double precision,boolean)

Version 3.2.1

  • Använda Boost: metrisk TSP ca

  • Simulated Annealing Algorithm stöds inte längre

    • Parametrar relaterade till Simulated Annealing Algorithm ignoreras: max_processing_time, tries_per_temperature, max_changes_per_temperature, max_consecutive_non_changes, initial_temperature, final_temperature, cooling_factor, randomize

Version 3.0.0

  • Namnändring från pgr_eucledianTSP

Version 2.3.0

  • Ny officiell funktion.

Beskrivning¶

Definition av problem¶

Problemet med den resande försäljaren (TSP) ställer följande fråga:

Givet en lista över städer och avstånden mellan varje par av städer, vilken är den kortaste möjliga rutten som besöker varje stad exakt en gång och återvänder till ursprungsstaden?

Egenskaper¶

  • Detta problem är ett NP-svårt optimeringsproblem.

  • Metrisk algoritm används

  • Implementeringen genererar lösningar som är dubbelt så långa som den optimala turen i värsta fall:

  • Grafiska egenskaper för bästa prestanda:

    • Grafen är ostyrd

    • Grafen är helt sammanhängande

    • Graf där resekostnader på kanter följer triangelolikheten.

    • Resekostnaderna är symmetriska:

      • Resekostnaderna från u till v är lika mycket som att resa från v till u

  • Resultaten ändras beroende på inmatningsordningen för Koordinater SQL

  • Alla duplicerade identifierare kommer att ignoreras. De koordinater som kommer att behållas är godtyckliga.

    • Koordinaterna är ganska lika för samma identifierare, till exempel

      1, 3.5, 1
1, 3.499999999999 0.9999999

    • Koordinaterna är helt olika för samma identifierare, till exempel

      2, 3.5, 1.0
2, 3.6, 1.1

Boost-graf inuti

Typ

Beskrivning

TSP valfria parametrar¶

Kolumn

Typ

Standard

Beskrivning

start_id

ANY-INTEGER

0

Det första besökande toppunktet

  • När 0 kan vilken toppunkt som helst bli den första besökande toppunkten.

slut_id

ANY-INTEGER

0

Sista besökande toppunkt innan återgång till start_vid.

  • När 0 kan vilken topp som helst bli den sista besökande toppen innan den återvänder till start_id.

  • När NOT 0 och start_id = 0 är det den första och sista toppunkten

Resultat kolumner¶

Returnerar SET OF (seq, nod, kostnad, agg_cost)

Kolumn

Typ

Beskrivning

seq

INTEGER

Radsekvens.

nod

BIGINT

Identifierare av nod/koordinat/punkt.

kostnad

FLOAT

Kostnad för att gå från aktuell nod till nästa nod i sökvägssekvensen.

  • 0 för den sista raden i turordningssekvensen.

agg_kostnad

FLOAT

Sammanlagd kostnad från noden vid seq = 1 till den aktuella noden.

  • 0 för den första raden i turordningen.

Total kostnad för resan¶

Få en total kostnad för turnén, jämför värdet med längden på en optimal turné är 27603, givet på datasetet

SELECT *
FROM pgr_TSPeuclidean($$SELECT * FROM wi29 ORDER BY id$$)
WHERE seq = 30;

-----+------+---------------+---------------
  30 |    1 | 2266.91173136 | 28777.4854127
(1 row)
