pgr_depthFirstSearch - Föreslagen¶

pgr_depthFirstSearch - Returnerar en genomgång av grafen med första sökdjupet. Grafen kan vara riktad eller oriktad.

Föreslagen

Varning

Föreslagna funktioner för nästa version av Mayor.

  • De finns inte officiellt med i den aktuella versionen.

    • Code has been reviewed therefore is not experimental.

    • Name, signature and functionality might not change.

    • pgTap-tester har gjorts. Men det kan behövas fler.

    • Dokumentationen kan behöva förbättras.

Tillgänglighet

Version 4.0.0:

  • Standardisering av utdatakolumner till (seq, depth, start_vid, pred, node, edge, cost, agg_cost)

Version 3.3.0

Version 3.2.0

  • Ny experimentell funktion.

Beskrivning¶

Depth First Search-algoritmen är en traverseringsalgoritm som startar från en rotvertex, går så djupt som möjligt och backar när en vertex nås utan några intilliggande vertex eller med alla besökta intilliggande vertex. Traversieringen fortsätter tills alla toppar som kan nås från rotvertexen har besökts.

De viktigaste egenskaperna är:

  • Implementationen fungerar för både directed och undirected grafer.

  • Ger traversieringsordningen Depth First Search från en rotvertex eller från en uppsättning rotvertex.

  • En valfri icke-negativ parameter för maximalt djup för att begränsa resultaten till ett visst djup.

  • För optimeringsändamål ignoreras alla duplicerade värden i Root vids.

  • Den producerar inte den kortaste vägen från en rotvertex till en målvertex.

  • Det är inte säkert att den sammanlagda kostnaden för traverseringen är minimal.

  • De returnerade värdena är ordnade i stigande ordning efter start_vid.

  • Djup Första sökning Löptid: \(O(E + V)\)

Parametrar¶

Parameter

Typ

Beskrivning

Kanter SQL

TEXT

Kanter SQL enligt beskrivningen nedan.

root vid

BIGINT

Identifierare för trädets rotvertex.

  • När värdet är \(0\) får du den spännvidd som börjar i aleatoriska noder för varje träd i skogen.

root vids

ARRAY [ ANY-TAL ]

Array med identifierare för rotpunkterna.

  • \(0\) värden ignoreras

  • För optimeringsändamål ignoreras alla duplicerade värden.

SMALLINT, INTEGER, BIGINT
