pgr_dijkstraVia¶

pgr_dijkstraVia - Rutt som går genom en lista med toppar.

Föreslagen

Varning

Föreslagna funktioner för nästa version av Mayor.

  • De finns inte officiellt med i den aktuella versionen.

    • Code has been reviewed therefore is not experimental.

    • Name, signature and functionality might not change.

    • pgTap-tester har gjorts. Men det kan behövas fler.

    • Dokumentationen kan behöva förbättras.

Tillgänglighet

Version 4.0.0

  • Funktion befordrad till tjänsteman.

Version 2.2.0

  • Ny föreslagen funktion.

Beskrivning¶

Med en lista över hörnpunkter och en graf är denna funktion likvärdig med att hitta den kortaste vägen mellan \(vertex_i\) och \(vertex_{i+1}\) för alla \(i < size\_of(via\;hörnpunkter)\).

Rutt:

är en sekvens av banor.

Sökväg:

är en del av rutten.

Boost-graf inuti

Underskrifter¶

En Via¶

pgr_dijkstraVia(Kanter SQL, via vertices, [options])
options: [directed, strict, U_turn_on_edge]
Returnerar uppsättning av (seq, path_id, path_seq, start_vid, end_vid, node, edge, cost, agg_cost, route_agg_cost)
ELLER TOM UPPSÄTTNING
Exempel:

Hitta den rutt som besöker topparna \(\{5, 1, 8\}\) i den ordningen i en riktad graf.

Resultat kolumner¶

Kolumn

Typ

Beskrivning

seq

INTEGER

Sekventiellt värde med början från 1.

Sökväg_id

INTEGER

Identifierare för ett led. Har värdet 1 för det första tågläget.

väg_seq

INTEGER

Relativ position i tågläget. Har värdet 1 för början av ett tågläge.

BIGINT

Identifierare för stigens startvertex.

slut_vid

BIGINT

Identifierare för banans slutpunkt.

nod

BIGINT

Identifierare av noden i sökvägen från start_vid till slut_vid.

kant

BIGINT

Identifierar den kant som används för att gå från node till nästa nod i bansekvensen.

  • -1 för den sista noden på vägen.

  • -2 för den sista noden i rutten.

cost

FLOAT

Kostnad för att gå från nod med hjälp av edge till nästa nod i vägsekvensen.

agg_cost

FLOAT

Sammanlagd kostnad från start_vid till node.

route_agg_cost

FLOAT

Total kostnad från start_vid av seq = 1 till slut_vid av den aktuella seq.

Ytterligare exempel¶

  • Den huvudsakliga frågeställningen

    • Sammanlagd kostnad för det tredje tågläget.

    • Ruttens sammanlagda kostnad för rutten vid slutet av den tredje vägen.

    • Noder som besökts i rutten.

    • De sammanlagda kostnaderna för rutten när de besökta topparna nås.

    • Status för ”passeringar framför” eller ”besök” av noderna.

Alla dessa exempel handlar om den rutt som besöker topparna \(\{5, 7, 1, 8, 15\}\) i den ordningen på en riktad graf.

Status för ”passeringar framför” eller ”besök” av noderna.¶

SELECT seq, route_agg_cost, node, agg_cost ,
  CASE WHEN edge = -1 THEN 'visits'
  ELSE 'passes in front'
  END as status
FROM  pgr_dijkstraVia(
  'SELECT id, source, target, cost, reverse_cost FROM edges order by id',
  ARRAY[5, 7, 1, 8, 15])
WHERE agg_cost  <> 0 or seq = 1;
