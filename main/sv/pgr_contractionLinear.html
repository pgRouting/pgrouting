pgr_contractionLinear - Föreslagen¶

pgr_contractionLinear - Utför grafkontraktion och returnerar de kontraherade topparna och kanterna.

Tillgänglighet

  • Version 3.8.0

    • Ny föreslagen funktion.

Beskrivning¶

Kontraktion minskar storleken på grafen genom att ta bort några av topparna och kanterna och till exempel lägga till kanter som representerar en sekvens av ursprungliga kanter, vilket minskar den totala tid och det utrymme som används i grafalgoritmer.

De viktigaste egenskaperna är:

  • Processen utförs endast på kanter med positiva kostnader.

  • Returnerar inte hela den kontraherade grafen.

    • Endast ändringar i diagrammet returneras.

  • De returnerade värdena inkluderar:

    • De nya kanterna som genereras av linjär sammandragning.

    • De modifierade topparna som genereras av sammandragning av återvändsgränder.

  • De returnerade värdena är ordnade enligt följande:

    • kolumn id stigande när det är en modifierad vertex.

    • kolumn id med negativa tal fallande när det är en ny kant.

Boost-graf inuti

Underskrifter¶

pgr_contractionLinear(Kanter SQL, [options])
options: [directed, forbidden]
Returnerar uppsättning av (type, id, contracted_vertices, source, target, cost)
Exempel:

Linjär sammandragning på en odelad graf.

SELECT * FROM pgr_contractionLinear(
  'SELECT id, source, target, cost, reverse_cost FROM edges',
  directed => false);
 type | id | contracted_vertices | source | target | cost
------+----+---------------------+--------+--------+------
 e    | -1 | {3}                 |      1 |      7 |    2
 e    | -2 | {17}                |     12 |     16 |    2
 e    | -3 | {15}                |     10 |     16 |    2
(3 rows)

  • De gröna noderna är linjära noder och kommer inte att vara en del av den kontrakterade grafen.

    • Alla angränsande kanter kommer inte att vara en del av den kontraherade grafen.

  • De röda linjerna kommer att vara nya kanter i den kontraherade grafen.

Linjära kanter¶

Oriktad graf

En nod förbinder två (eller fler) linjära kanter när

  • Antalet angränsande hörn är 2.

I en riktad graf anses en nod vara en ”linjär” nod när

  • Antalet angränsande hörn är 2.

  • Linjäriteten är symmetrisk.

Linjäriteten är inte symmetrisk¶

Riktad graf

Graf där linjäriteten inte är symmetrisk.

När grafen bearbetas som en riktad graf är linjäriteten inte symmetrisk, och grafen kan därför inte dras ihop.

SELECT * FROM pgr_contractionLinear(
$$SELECT * FROM (VALUES
  (1, 1, 2, 1, -1),
  (2, 2, 3, 3, 4))
  AS edges(id,source,target,cost,reverse_cost)$$,
  directed => true);

------+----+---------------------+--------+--------+------
(0 rows)

Steg för steg linjär sammandragning¶

Den linjära sammandragningen upphör när det inte längre finns några linjära kanter. Till exempel från följande graf finns det linjära kanter

Contracting vertex \(3\),

  • Toppunkten \(3\) tas bort från grafen

  • Kanterna \(2 \rightarrow 3\) och \(w \rightarrow z\) tas bort från grafen.

  • En ny kant \(2 \rightarrow 4\) infogas representerad med röd färg.

Kontraherande vertex \(2\):

  • Toppunkten \(2\) tas bort från grafen

  • Kanterna \(1 \rightarrow 2\) och \(2 \rightarrow 3\) tas bort från grafen.

  • En ny kant \(1 \rightarrow 3\) infogas representerad med röd färg.

Edge \(1 \rightarrow 3\) har information om kostnad och de noder som kontrakterades.

SELECT * FROM pgr_contractionLinear(
$$SELECT * FROM (VALUES
  (1, 1, 2, 1),
  (2, 2, 3, 1),
  (2, 3, 4, 1))
  AS edges(id,source,target,cost)$$);
 type | id | contracted_vertices | source | target | cost
------+----+---------------------+--------+--------+------
 e    | -1 | {2,3}               |      1 |      4 |    3
(1 row)

Steg för att skapa den avtalade grafen¶

Lägg till ytterligare kolumner.

ALTER TABLE vertices ADD is_contracted BOOLEAN DEFAULT false;
ALTER TABLE
ALTER TABLE edges ADD is_new BOOLEAN DEFAULT false;
ALTER TABLE
ALTER TABLE edges ADD contracted_vertices BIGINT[];
ALTER TABLE

Spara resultaten i en tabell.

SELECT * INTO contraction_results
FROM pgr_contractionLinear(
  'SELECT id, source, target, cost, reverse_cost FROM edges',
  directed => false);
SELECT 3

Använd kolumnen is_contracted för att ange de toppar som är kontraherade.

UPDATE vertices
SET is_contracted = true
WHERE id IN (SELECT  unnest(contracted_vertices) FROM  contraction_results);
UPDATE 3

De kontraherade topparna är inte en del av den kontraherade grafen.

SELECT id, is_contracted
FROM vertices WHERE is_contracted ORDER BY id;
 id | is_contracted
----+---------------
  3 | t
 15 | t
 17 | t
(3 rows)

Infoga de nya kanterna som genererats av pgr_contraction.

INSERT INTO edges(source, target, cost, reverse_cost, contracted_vertices, is_new)
SELECT source, target, cost, -1, contracted_vertices, true
FROM contraction_results;
INSERT 0 3

Skapa den kontraherade grafen.

CREATE VIEW contracted_graph AS
WITH
vertices_in_graph AS (
  SELECT id FROM vertices WHERE NOT is_contracted
)
SELECT id, source, target, cost, reverse_cost
FROM edges
WHERE source IN (SELECT * FROM vertices_in_graph)
AND target IN (SELECT * FROM vertices_in_graph)
ORDER BY id;
CREATE VIEW

Används när avgång/destination inte finns i den avtalade grafen¶

SELECT * FROM pgr_dijkstra(
  'WITH in_line AS (SELECT contracted_vertices FROM edges WHERE 17 = ANY(contracted_vertices))
   SELECT id, source, target, cost, reverse_cost
   FROM edges, in_line
   WHERE source = ANY(in_line.contracted_vertices) OR target = ANY(in_line.contracted_vertices)

  UNION

  SELECT id, source, target, cost, reverse_cost FROM contracted_graph',
  1, 17);
