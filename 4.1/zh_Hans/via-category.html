一般信息¶

该类别旨在解决普遍问题：

给定一个图和一个顶点列表，找到所有相邻顶点对 \(vertex_i\) 和 \(vertex_{i+1}\) 之间的最短路径

换句话说，寻找一条连续路径，使其按照给定顺序访问所有节点。

path:

代表 route 的一部分。

route:

是一系列 paths

CHAR

值在 [r, R, l, L, b, B] 中表示驾驶的一侧是：

  • For directed graph:

    • [r, R] for right driving side

    • [l, L] for left driving side

  • For undirected graph:

    • [b, B] for both driving sides

  • Default values when parameter is missing:

    • [r, R] for directed graph

    • [l, L] for undirected graph

Via可选参数¶

用于所有 Via 功能

参数

类型

默认

描述

strict

BOOLEAN

false

  • 当为 true 时，如果路径丢失，则停止并返回 EMPTY SET

  • 当 false 忽略丢失的路径时，返回找到的所有路径

U_turn_on_edge

BOOLEAN

true

  • 为 true 时，从访问过的顶点出发时不会试图避开

Points SQL¶

用于

  • pgr_withPointsVia

参数

类型

默认

描述

pid

ANY-INTEGER

value

点的标识符。

  • 使用正值，因为内部将转换为负值

  • 如果列存在，则它不能为 NULL。

  • 如果列不存在，将自动给出连续的负 值 。

edge_id

ANY-INTEGER

距离该点“最近”的边的标识符。

fraction

ANY-NUMERICAL

Value in <0,1> that indicates the relative position from the first end point of the edge.

side

CHAR

b

[b, r, l, NULL] 中的值指示该点是否为：

  • r 在右边，

  • l 在左边，

  • b, NULL 在两边

其中：

ANY-INTEGER:

SMALLINT, INTEGER, BIGINT

ANY-NUMERICAL:

SMALLINT, INTEGER, BIGINT, REAL, FLOAT

结果列¶

列

类型

描述

seq

INTEGER

从 1 开始的顺序值。

path_id

INTEGER

路径的标识符。 第一条路径的值为 1。

path_seq

INTEGER

路径中的相对位置。 路径开头的值为 1 。

start_vid

BIGINT

路径起始顶点的标识符。

end_vid

BIGINT

路径结束顶点的标识符。

node

BIGINT

从 start_vid 到 end_vid 路径中节点的标识符。

edge

BIGINT

用于从路径序列中的 node 到下一个节点的边的标识符。

  • -1，表示路径的最后一个节点。

  • 路由的最后一个节点为 -2。

cost

FLOAT

从使用 edge 的 node 遍历到路径序列中的下一个节点的成本。

agg_cost

FLOAT

从 start_vid 到 node 的总成本。

route_agg_cost

FLOAT

从 seq = 1 的 start_vid 到当前 seq 的 end_vid 的总费用。

备注

当 start_vid 、 end_vid 和 node 列具有负值时，该标识符用于Point。
