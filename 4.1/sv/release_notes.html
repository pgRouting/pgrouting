Skapa¶

  • C++-standarden är std17

    • Med hjälp av denna standard kommer alla verktygskedjor som stöds i vår CI-matris att fungera.

    • Koden är ännu inte modifierad för att använda std17:

      • Om det behövs: -DCMAKE_CXX_STANDARD=14 för att sänka standarden.

  • Användarens dokumentation byggs som standard.

  • Doxygen-dokumentationen byggs som standard.

För utvecklare:

  • Ställ in -DUSE_CLANG_TIDY=ON för clang tidy-kontroller.

  • Ordentliga kontroller görs på CI.

Uppbyggnad av dokumentation¶

  • Doxygen-dokumentationen byggs som standard

  • HTML-dokumentationen byggs som standard

  • HTML-dokumentationen för översatta språk (en, es, sv, zh_Hans) byggs som standard

  • WITH-DOC används inte längre

Användardokumentation skapas inte när

  • Sfinxen hittas inte

  • När alla Sphinx-format är OFF

    • För att inte bygga HTML-standardformat: -DBUILD_HTML=OFF

  • När alla språk är OFF

    • Om du bara vill bygga på engelska: -DES=OFF -DSV=OFF -DZH_HANS=OFF

  • Plats för dokumentationsutdata: build/doc/_build/<format>

    • Till exempel: för HTML-utdata i katalogen build/doc/_build/html

Dokumentation för utvecklare byggs inte när

  • Doxygen hittas inte

  • Om du inte vill bygga Doxygen-dokumentation: -DBUILD_DOXY=OFF

Versionsinformation för pgRouting 2.0.0¶

För att se alla problem och pull requests som stängts av denna release, se Github milstolpe för 2.0.0

Med lanseringen av pgRouting 2.0.0 har biblioteket övergett bakåtkompatibilitet till pgRouting 1.0-utgåvor. Huvudmålen för denna utgåva är:

  • Stor omstrukturering av pgRouting.

  • Standardisering av namngivning av funktioner

  • Förberedelse av projektet för framtida utveckling.

Som ett resultat av denna insats:

  • pgRouting har en förenklad struktur

  • Betydande ny funktionalitet har lagts till

  • Dokumentation har integrerats

  • Testning har integrerats

  • Och gjort det enklare för flera utvecklare att bidra.

Viktiga förändringar

  • Graph Analytics - verktyg för att upptäcka och åtgärda vissa problem med kopplingar i en graf

  • En samling användbara nyttofunktioner

  • Två nya algoritmer för korta vägar för alla par (pgr_apspJohnson, pgr_apspWarshall)

  • Dubbelriktade Dijkstra- och A-star-sökalgoritmer (pgr_bdAstar, pgr_bdDijkstra)

  • Sökning från en till många noder (pgr_kDijkstra)

  • K alternativa vägar kortaste vägen (pgr_ksp)

  • Ny TSP-lösare som förenklar koden och byggprocessen (pgr_tsp), har tagit bort beroendet av ”Gaul Library”

  • Turn Begränsad kortaste väg (pgr_trsp) som ersätter Shooting Star

  • Slopat stöd för Shooting Star

  • Byggt upp en testinfrastruktur som körs innan större kodändringar checkas in

  • Testat och åtgärdat de flesta av de utestående buggarna som rapporterats mot 1.x och som finns i 2.0-dev-kodbasen.

  • Förbättrad byggprocess för Windows

  • Automatiserad testning på Linux- och Windows-plattformar som utlöses av varje åtagande

  • Modulär biblioteksdesign

  • Kompatibilitet med PostgreSQL 9.1 eller nyare

  • Kompatibilitet med PostGIS 2.0 eller nyare

  • Installeras som PostgreSQL EXTENSION

  • Returtyperna är faktorerade och enhetliga

  • Stöd för tabellen SCHEMA i funktionsparametrar

  • Stöd för PostGIS-funktionsprefixet st_

  • Lagt till prefixet pgr_ för funktioner och typer
