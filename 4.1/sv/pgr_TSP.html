pgr_TSP¶

  • pgr_TSP - Approximation med hjälp av metrisk algoritm.

Tillgänglighet:

Version 4.0.0

  • Resultaten ändras beroende på inmatningsordning

  • Endast för odelade grafer

  • Förändring på gång, signaturer inte längre tillgängliga:

    • pgr_tsp(text,bigint,bigint,double precision,integer,integer,integer,double precision,double precision,double precision,boolean)

Version 3.2.1

  • Metrisk algoritm från Boost-biblioteket

  • Simulated Annealing Algorithm stöds inte längre

    • Parametrar relaterade till Simulated Annealing Algorithm ignoreras: max_processing_time, tries_per_temperature, max_changes_per_temperature, max_consecutive_non_changes, initial_temperature, final_temperature, cooling_factor, randomize

Version 2.3.0

  • Ändring av signatur

  • Gammal signatur stöds inte längre

Version 2.0.0

  • Officiell funktion.

Beskrivning¶

Definition av problem¶

Problemet med den resande försäljaren (TSP) ställer följande fråga:

Givet en lista över städer och avstånden mellan varje par av städer, vilken är den kortaste möjliga rutten som besöker varje stad exakt en gång och återvänder till ursprungsstaden?

Egenskaper¶

  • Detta problem är ett NP-svårt optimeringsproblem.

  • Metrisk algoritm används

  • Implementeringen genererar lösningar som är dubbelt så långa som den optimala turen i värsta fall:

  • Grafiska egenskaper för bästa prestanda:

    • Grafen är ostyrd

    • Grafen är helt sammanhängande

    • Graf där resekostnader på kanter följer triangelolikheten.

    • Resekostnaderna är symmetriska:

      • Resekostnaderna från u till v är lika mycket som att resa från v till u

  • Resultaten ändras beroende på inmatningsordningen för Matrix SQL

  • Negativa kostnader ignoreras.

  • Kan användas med Kostnadsmatris - Kategori funktioner helst med directed => false.

    • Med riktad => falsk

      • Kommer att generera en graf som:

        • är ostyrd

        • är helt sammanhängande (så länge grafen har en komponent)

        • alla resekostnader på kanter följer triangelolikheten.

      • När start_vid = 0 OR end_vid = 0

        • De lösningar som genereras är garanterat * dubbelt så långa som den optimala turen i värsta fall*

      • När start_vid != 0 AND end_vid != 0 AND start_vid != end_vid

        • Det är inte garanterat att lösningen i värsta fall kommer att vara dubbelt så lång som den optimala turen, eftersom end_vid tvingas vara i en fast position.

    • Med riktad => sann

      • Det är inte garanterat att lösningen i värsta fall kommer att vara dubbelt så lång som den optimala turen

      • Kommer att generera en graf som:

        • är riktad

        • är helt sammanhängande (så länge grafen har en komponent)

        • vissa (eller alla) resekostnader på kanter kanske inte följer triangelolikheten.

      • Eftersom det krävs en oriktad graf, omvandlas den riktade grafen enligt följande:

        • kanterna (u, v) och (v, u) anses vara samma kant (betecknas (u, v))

        • om agg_cost skiljer sig mellan en eller flera instanser av kant (u, v)`

        • Det lägsta värdet av agg_cost för alla instanser av kanten (u, v)` kommer att betraktas som agg_cost för kanten (u, v)`

        • Vissa (eller alla) resekostnader på kanter kommer kanske ändå inte att följa triangelolikheten.

  • När data inte kommer från en odelad graf eller om den inte är helt sammanhängande:

    • Saknade värden kommer att beräknas med dijkstra-algoritmen.

    • När grafen har mer än en komponent:

      • start_vid eller slut_vid är definierade och finns på samma komponent: TSP-turen kommer att äga rum på den komponenten.

      • start_vid eller slut_vid definieras och inte finns på samma komponent: TSP-turen kommer att föreslå en tur som har båda komponenterna där anslutningskostnaderna beräknas.

      • start_vid eller end_vid är inte definierade: startpunkten kan vara på vilken komponent som helst och kommer endast att innehålla data från den komponenten.

  • Ett cykelförsök för att ta bort korsande kanter görs på TSP-resultaten.

Parametrar¶

Parameter

Typ

Beskrivning

TSP valfria parametrar¶

Kolumn

Typ

Standard

Beskrivning

start_id

ANY-INTEGER

0

Det första besökande toppunktet

  • När 0 kan vilken toppunkt som helst bli den första besökande toppunkten.

end_id

ANY-INTEGER

0

Sista besökande toppunkt innan återgång till start_vid.

  • När 0 kan vilken topp som helst bli den sista besökande toppen innan den återvänder till start_id.

  • När NOT 0 och start_id = 0 är det den första och sista toppunkten

Kolumn

Typ

Beskrivning

node

BIGINT

Identifierare av nod/koordinat/punkt.

cost

FLOAT

Kostnad för att gå från aktuell nod till nästa nod i sökvägssekvensen.

  • 0 för den sista raden i turordningssekvensen.

agg_cost

FLOAT

Sammanlagd kostnad från noden vid seq = 1 till den aktuella noden.

  • 0 för den första raden i turordningen.

Använda intressepunkter för att generera en asymmetrisk matris.¶

För att generera en asymmetrisk matris:

  • Linje 4 Informationen om sidan av pointsOfInterset ignoreras genom att inte inkludera den i frågan

  • Linje 6 Generering av en asymmetrisk matris med directed => true

    • \(min(agg\_cost(u, v), agg\_cost(v, u))\) is going to be considered as the agg_cost

    • Lösningen som kan vara större än dubbelt så lång som den optimala turen eftersom:

      • Triangelns ojämlikhet kanske inte uppfylls.

      • start_id != 0 AND end_id != 0

 1SELECT * FROM pgr_TSP(
 2  $$SELECT * FROM pgr_withPointsCostMatrix(
 3    'SELECT id, source, target, cost, reverse_cost FROM edges ORDER BY id',
 4    'SELECT pid, edge_id, fraction from pointsOfInterest',
 5    array[-1, 10, 7, 11, -6], 'r',
 6    directed => true) $$,
 7  start_id => 7,
 8  end_id => 11);
